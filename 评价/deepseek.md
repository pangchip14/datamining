## 二、主要风险与问题（不解决无法拿 A）

### 🔴 风险1：**数据规模“20–40条序列”可能偏小，削弱结论的说服力**

- 课程要求明确写：**“evaluated on many datasets”** 才能拿更好成绩。  
- TSB-UAD Public-v2 中单变量数据集远多于 40 条。如果你只挑 20 条，会被质疑“cherry-picking”。  
- **建议**：至少跑 **50–80 条序列**，哪怕每条序列较短。数量本身在作业评分中很重要。

### 🔴 风险2：**Matrix Profile 的实现与公平比较存在隐患**

- Matrix Profile 通常输出的是 **discord score**（不相似度），而其他算法输出的是异常分数。  
- 分数分布、阈值敏感度差异很大。你在方案中没有说明如何做 **score alignment** 到点级。  
- 如果实现不严谨，Matrix Profile 可能会明显吃亏或被不公正地抬高分。  
- **建议**：明确写出“Matrix Profile 使用 SCRIMP++ 或 STOMP，点级分数取窗口中心对齐”。

### 🟡 风险3：**“解释型研究”在 final report 中很容易沦为口头承诺**

- 你现在的方案写的是“要解释为什么”，但最终评分看的是 **report 中是否真的做到了**。  
- 常见失败模式：  
  - 结果出来后只说“kNN 在长序列上表现差”，但没有分析 window embedding 维度灾难。  
  - 只说“LOF 对密度变化敏感”，但没有用数据集的具体特征（如局部密度方差）验证。  
- **建议**：提前设计一个 **failure analysis 模板**——对每个算法，列出 2–3 个它应失败的条件，然后在实验中有目的地选择包含这些条件的数据集。

### 🟡 风险4：**作业页数限制（≤10页正文）与你的内容量之间存在冲突**

- 你的方案内容量（6类算法 × 多指标 × 消融 × 多数据集 × 解释分析）很容易写到 15–20 页。  
- 超过 10 页会被 cut，而且扣分。  
- **建议**：  
  - 主结果用表格 + 热力图 + 散点图，不要大段文字。  
  - 每个 RQ 对应 1–2 个核心图，附录放完整结果（但附录不算分）。

### 🟡 风险5：**与已有工作的区别不够尖锐**

- 你说“区别于 TSB-UAD / TimeEval”，但 TSB-UAD 本身已经做了多数据集、多算法比较。  
- 你的真正区别应该是：**不追求覆盖率，而是做“数据特征 → 算法机制”的因果解释**。需要在 related work 中更直接地批评现有 benchmark 不解释原因。  
- 否则 reviewer（助教/老师）会认为“这本质上还是一个 benchmark”。

## 三、关于 A 级分数的具体判断标准

根据 assignment 文件第 4 页：

> An A grade report is a succinct and well-organized report which describes the process in achieving the clearly stated aim, and provides an **insightful analytical result**.

关键词是 **insightful**。这意味着：

- 你的分析要能回答 **“为什么”**，而不是“是什么”。  
- 要有一条清晰的故事线，比如：  
  > “我们发现 Isolation Forest 在所有数据集上排名中等，但进一步分析显示：当异常表现为局部密集区域中的孤立点时，IF 表现最佳；当异常是全局形状异常时，IF 几乎完全失效。这与随机划分机制的本质一致。”

  如果最终报告只是给出：
- Table 1: 平均排名  
- Figure 1: Runtime vs AUPRC  
- 然后说“kNN 总体上最好”

那就是 B / B+，不是 A。
