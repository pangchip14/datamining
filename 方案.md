# 时间序列异常检测算法比较：完整研究方案

## 1. Summary

本项目选择 **univariate time-series anomaly detection（单变量时间序列异常检测）**，目标不是简单复现算法排行榜，而是做一个 **explanatory empirical study（解释型实证研究）**：比较多类代表性算法在多个时间序列数据集上的 accuracy（准确性）、robustness（鲁棒性）和 computational efficiency（计算效率），并进一步解释不同数据特征下算法表现差异的原因。

**Aim Statement**

> The aim of this study is to evaluate and explain the accuracy, robustness, and computational efficiency of representative unsupervised anomaly detection algorithms for univariate time-series data across datasets with different anomaly characteristics.

中文解释：本研究旨在比较并解释多种无监督异常检测算法在单变量时间序列数据上的准确性、鲁棒性和计算效率，重点分析它们在不同异常特征的数据集上为何表现不同。

## 2. Research Questions 与 Motivation

**Motivation（研究动机）**

已有 benchmark（基准评测）通常给出算法总体排名，但实际应用中更关键的问题是：某个算法在什么类型的数据上有效、什么时候失败、为什么失败，以及它的效果是否值得计算成本。因此本研究不仅比较“哪个算法最好”，还要分析“什么时候好、为什么好”。

**Research Questions（研究问题）**

- RQ1：哪些 unsupervised anomaly detection algorithms（无监督异常检测算法）在多个 univariate time-series datasets（单变量时间序列数据集）上整体表现最好？
- RQ2：算法表现是否受 anomaly ratio（异常比例）、series length（序列长度）、anomaly duration（异常持续时间）、trend/seasonality（趋势/季节性）等 dataset characteristics（数据特征）影响？
- RQ3：不同算法在 accuracy（准确性）和 computational efficiency（计算效率）之间是否存在明显 trade-off（权衡）？
- RQ4：某些算法在哪些数据条件下容易失败？这些 failure modes（失败模式）是否可以由 algorithm mechanism（算法机制）解释？

**与已有工作的区别**

本项目会引用 TSB-UAD、TimeEval、VUS metrics（时间序列异常检测评价指标）和代表性算法论文。区别在于：本研究不追求覆盖最多算法，而是围绕数据特征和算法机制做可解释比较，强调 performance ranking + sensitivity analysis + efficiency analysis + failure analysis。

## 3. Experimental Design

**Data Scope（数据范围）**

主数据源使用 **TSB-UAD Public-v2**。若时间允许，可加入 **UCR Time Series Anomaly Datasets 2021** 作为 external validation（外部验证）。

默认实验规模：

- 选取 20-40 条 labeled univariate time series（带异常标签的单变量时间序列）。
- 覆盖不同 anomaly ratio（异常比例）、series length（序列长度）和 anomaly duration（异常持续时间）。
- 优先选择包含不同 pattern type（模式类型）的序列，如趋势型、周期型、噪声型、形态变化型。
- 固定 random seed = 20260423，保证 reproducibility（可复现性）。

**Algorithms（算法）**

主实验比较 6 类方法：

- Rolling Z-Score：滑动窗口标准分，simple statistical baseline（简单统计基线）。
- kNN Detector：基于 nearest-neighbor distance（最近邻距离）的异常检测。
- LOF：Local Outlier Factor，基于 local density（局部密度）的异常检测。
- Isolation Forest：基于 random partition / isolation mechanism（随机划分/隔离机制）的树模型。
- One-Class SVM：学习 normal boundary（正常边界）的 boundary-based detector（边界型检测器）。
- Matrix Profile：用于发现 discord subsequence（不相似子序列）的时间序列专用方法。

不把 deep learning（深度学习）列为主实验。原因是月底前完成工作时，深度模型训练成本和调参成本较高，且多数据集上公平比较困难。可在 related work 或 discussion 中说明这一取舍。

**Methodology Pipeline（方法流程）**

统一流程如下：

1. Data loading（数据加载）：读取 time series values 和 anomaly labels。
2. Preprocessing（预处理）：处理 missing values（缺失值）、非法值，并进行 z-normalization（标准化）。
3. Sliding-window embedding（滑动窗口嵌入）：将时间点序列转换为窗口向量，供 kNN、LOF、Isolation Forest、One-Class SVM 使用。
4. Model scoring（模型打分）：每个算法输出 anomaly score（异常分数）。
5. Score alignment（分数对齐）：将窗口级分数映射回时间点级分数。
6. Evaluation（评价）：labels 只在评价阶段使用，不参与无监督训练。
7. Analysis（分析）：按算法、数据集特征、指标和运行时间分组分析。

默认 window size（窗口长度）设为：

> w = max(16, min(128, floor(0.02 × n)))

其中 n 是序列长度。

## 4. Evaluation 与 Ablation Studies

**Evaluation Methodology（评价方法）**

采用 **unsupervised setting（无监督设置）**：

- 训练或打分阶段不使用 anomaly labels。
- labels 只用于最终评价。
- 若数据集提供 normal training segment（正常训练片段），可在 optional ablation 中比较其影响；主实验默认使用 full-series unsupervised scoring（整条序列无监督打分）。

**Metrics（评价指标）**

主指标：

- AUROC：Area Under ROC Curve，衡量整体排序能力。
- AUPRC：Area Under Precision-Recall Curve，更适合异常比例低的数据。
- VUS-ROC：Volume Under Surface ROC，考虑时间序列异常范围容忍度的 ROC 指标。
- VUS-PR：Volume Under Surface PR，考虑时间序列异常范围容忍度的 PR 指标。
- Runtime：运行时间，用于衡量计算效率。
- Average Rank：跨数据集平均排名。

辅助指标：

- Best F1：使用事后最优阈值的 F1，仅作辅助，不作为核心结论。
- Contamination-based F1：按已知异常比例设阈值的 F1，若时间允许使用。

**Core Ablation Studies（必做消融实验）**

- Window Size Sensitivity：比较 window size = 1%、2%、5% series length，观察窗口大小对 kNN、LOF、Isolation Forest、One-Class SVM 的影响。
- Normalization Effect：比较 raw values（原始值）和 z-normalized values（标准化值），重点分析 kNN、LOF、One-Class SVM 等对尺度敏感的方法。
- Metric Choice Effect：比较 AUROC、AUPRC、VUS-PR 下的算法排名是否一致，说明不同评价指标会如何改变结论。

**Optional Ablation Studies（选做消融实验）**

- Training Setting Effect：如果数据支持，比较 normal training segment 和 full-series scoring 的结果差异。
- Thresholding Strategy Effect：比较 best F1 threshold 和 contamination-based threshold，说明阈值策略是否影响结论。
- Algorithm Family Removal：从 discussion 角度分析不同算法家族是否贡献互补能力，不一定单独跑大量实验。

## 5. Analysis 与 Presentation/Report 输出

**原因解释框架**

结果分析要结合 algorithm mechanism（算法机制）：

- Rolling Z-Score：适合 sudden point anomaly（突发点异常），但对 shape anomaly（形状异常）较弱。
- kNN：如果异常窗口远离正常窗口，效果较好；但可能受 high-dimensional window representation（高维窗口表示）和 noise（噪声）影响。
- LOF：适合 local density anomaly（局部密度异常），但正常数据密度变化大时容易误判。
- Isolation Forest：通常效率较好，但如果异常窗口不容易被随机划分隔离，表现会下降。
- One-Class SVM：可能在结构清晰的数据上有效，但对 kernel parameter（核参数）和数据规模敏感。
- Matrix Profile：适合 subsequence anomaly（子序列异常）或 shape anomaly，但对点异常或标签粒度很细的异常可能不稳定。

**Preparation Presentation 内容**

12 分钟 presentation 建议 10-12 页：

- Title：题目、组员、项目方向。
- Motivation：为什么不仅要比较算法，还要解释表现差异。
- Aim Statement：展示研究目标。
- Research Questions：RQ1-RQ4。
- Related Work：TSB-UAD、TimeEval、VUS、代表性算法。
- Dataset Scope：数据来源与选择规则。
- Algorithms：6 类算法及选择理由。
- Methodology Pipeline：从 preprocessing 到 scoring 到 evaluation。
- Evaluation Metrics：AUROC、AUPRC、VUS-ROC、VUS-PR、Runtime。
- Ablation Studies：3 个必做消融实验和可选消融。
- Expected Analysis：ranking、sensitivity、trade-off、failure analysis。
- Risks and Controls：工作量控制、复现性、避免调参不公平。

**Final Report 重点**

最终报告严格围绕 aim，不写无关内容。建议重点图表包括：

- Overall performance ranking table。
- Metric-specific ranking comparison。
- Runtime vs AUPRC scatter plot。
- Performance grouped by anomaly ratio / series length / anomaly duration。
- Window size ablation plot。
- Normalization ablation plot。
- 2-3 个 case studies，展示算法成功和失败的具体序列。

## 6. Assumptions 与取舍

- 主任务限定为 univariate time-series anomaly detection。
- 主实验采用 unsupervised setting。
- 主数据集使用 TSB-UAD Public-v2，UCR 作为可选外部验证。
- 主算法为 Rolling Z-Score、kNN、LOF、Isolation Forest、One-Class SVM、Matrix Profile。
- 主指标为 AUROC、AUPRC、VUS-ROC、VUS-PR、Runtime、Average Rank。
- 必做 3 个消融实验：window size、normalization、metric choice。
- 不把 deep learning 放入主实验，避免月底前工作量失控和公平调参问题。
- 两人团队优先保证实验质量、分析深度和报告逻辑，而不是盲目扩大算法或数据集数量。
