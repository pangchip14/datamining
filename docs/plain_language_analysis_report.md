# 通俗易懂版详细分析报告

日期：2026-04-28  
主题：用简单语言解释我们的时间序列异常检测项目

---

## 1. 我们到底在做什么

我们研究的是时间序列异常检测。

时间序列就是按时间顺序记录的一串数字。例如：

- 每分钟的机器温度
- 每秒的网络流量
- 每天的股票价格
- 每小时的用电量
- 医疗监测中的心电信号

异常检测就是在这些数字里找“不正常”的片段。比如机器突然升温、网络流量突然暴涨、传感器读数出现奇怪波动。

我们的项目只研究单变量时间序列，也就是每条数据只有一个数值通道。例如只看温度，不同时看温度、湿度、压力。

---

## 2. 为什么这个问题不简单

异常检测看起来像是“找特别大的数”，但实际更复杂。

有些异常确实是突然变大或变小，例如一个尖峰。Rolling Z-Score 这种简单方法就能处理。

但很多异常不是这样：

- 数值没有特别大，只是形状变了。
- 单个点看起来正常，但连续一段组合起来不正常。
- 在一个时间段里正常的值，放到另一个时间段就不正常。
- 数据本身有周期性，异常藏在周期变化里面。
- 噪声很大，正常波动和异常很难分开。

所以我们不能只用一个简单规则，而要比较多种算法。

---

## 3. 我们比较了哪些算法

我们一共比较六类方法。

### 1. Rolling Z-Score

它的想法最简单：看当前点和附近平均值差多少。

如果一个点比附近点大很多或小很多，就认为它异常。

优点：

- 很快
- 容易解释

缺点：

- 只擅长找尖峰
- 不擅长找形状变化
- 不擅长找连续片段异常

所以它更像一个 baseline，用来告诉我们“简单统计规则能做到什么程度”。

### 2. kNN

kNN 的想法是：把每一小段时间序列当作一个形状，然后找相似形状。

如果某一段和其他大多数片段都不像，它就可能是异常。

它的优点是很直观：异常就是“不像别人”的片段。

我们的结果里，kNN 是整体最好的算法。

### 3. LOF

LOF 看的是局部密度。

通俗说，它不只是问“你离别人远不远”，还问“你周围是不是比别人的周围更稀疏”。

如果一个片段周围很空，而附近其他片段都处在比较密集的区域，它就更可能异常。

LOF 有时表现很好，但对参数和重复值比较敏感。实验中我们也看到 sklearn 给出 duplicate values warning，这说明某些数据里的重复值会影响它的局部密度判断。

### 4. Isolation Forest

Isolation Forest 的直觉是：异常点更容易被隔离。

如果通过随机切分数据，很快就能把某个点或片段单独分出来，那它可能比较异常。

它的优势是速度和稳定性。虽然它不是最高分，但它是最均衡的方法之一。

### 5. IDKD

IDKD 是 Isolation Distributional Kernel Detector，属于 Isolation Kernel family。我们使用的是 `ikpykit.anomaly.IDKD` 官方实现。

它比我们之前的轻量版本更严格，也更符合课程里 Isolation Kernel 相关内容。

不过完整官方实现更慢。它在主实验中平均每条序列约 3.10 秒，比其他方法慢很多。

它的结论不是“没用”，而是：它有机制价值和课程相关性，但在我们当前的滑动窗口设置下，不是最经济的默认方法。

### 6. Matrix Profile

Matrix Profile 是时间序列专用方法。

它会问：每一段时间序列有没有相似的另一段。如果某段找不到相似片段，它就可能异常。

它适合找罕见形状，但如果异常不是以“罕见形状”的方式出现，或者窗口大小不合适，它就不一定强。

---

## 4. 我们怎么保证比较公平

为了公平，我们让所有算法尽量走同一套流程：

1. 读取同一批数据。
2. 使用同样的数据清洗。
3. 默认都做 z-normalization。
4. 用同样的滑动窗口方法把时间序列切成小段。
5. 每个算法对这些小段打异常分数。
6. 把窗口分数映射回每个时间点。
7. 用同一套指标评价。

这里有一个关键点：标签只在最后评价时使用，不参与算法训练、参数选择，也不参与长序列裁剪。

这可以避免“提前偷看答案”。

---

## 5. 我们用什么指标判断好坏

异常检测不能只看 accuracy。因为异常通常很少，如果 99% 都是正常点，一个什么都不报的模型也可能有很高 accuracy，但它没有实际价值。

我们主要看 range-aware VUS-PR。

可以简单理解为：

- PR 更关注异常点有没有被找出来。
- VUS 允许时间序列异常有一点范围容忍度，不要求每个点完全对齐。

这比普通 accuracy 更适合时间序列异常检测。

我们现在用的是和 TSB-UAD / TSB-AD 公开实现逻辑一致、并经过官方核心代码交叉核对的 VUS，不再用早期的近似版本作为主指标。

---

## 6. 主实验结果

我们在 60 条真实时间序列上比较了 6 个算法。

按 VUS-PR 排序：

| Algorithm | Mean VUS-PR | Average Rank | Runtime |
|---|---:|---:|---:|
| kNN | 0.4929 | 2.45 | 0.41s |
| Isolation Forest | 0.4577 | 3.11 | 0.26s |
| LOF | 0.4416 | 3.17 | 0.43s |
| IDKD | 0.3986 | 3.75 | 3.10s |
| Matrix Profile | 0.3599 | 4.18 | 0.45s |
| Rolling Z-Score | 0.2768 | 4.34 | 0.14s |

最重要的结论：

- kNN 整体最好。
- Isolation Forest 最均衡。
- LOF 有竞争力，但稳定性略弱。
- IDKD 更严格、更官方，但运行时间高。
- Matrix Profile 有专门场景价值。
- Rolling Z-Score 很快，但效果最弱。

---

## 7. 为什么 kNN 会赢

kNN 赢的原因和时间序列异常的形式有关。

我们把时间序列切成很多小窗口。每个窗口代表一段局部形状。

如果异常表现为“这一段形状不像其他段”，kNN 很自然就能发现。因为它会计算这个窗口和其他窗口的距离。

很多真实异常并不只是单点尖峰，而是局部模式变化。所以 kNN 这种基于窗口形状的距离方法会很强。

---

## 8. 为什么 Isolation Forest 很重要

虽然 Isolation Forest 的平均 VUS-PR 不是第一，但它有两个优点：

1. 速度快。
2. 稳定。

主实验中它平均运行时间约 0.26 秒，明显快于 kNN、LOF、Matrix Profile，更远快于 IDKD。

在异常比例较高、异常持续时间较长、噪声较强的数据上，它经常比 kNN 更稳。

所以如果实际应用中既想要不错效果，又在意运行成本，Isolation Forest 是很值得考虑的选择。

---

## 9. 标准化为什么重要

标准化就是把数据变成均值约为 0、标准差约为 1 的形式。

这对距离型方法很重要。

真实消融结果显示：

| Algorithm | Raw VUS-PR | Z-normalized VUS-PR |
|---|---:|---:|
| kNN | 0.3681 | 0.3709 |
| IDKD | 0.2151 | 0.3265 |
| Isolation Forest | 0.3383 | 0.3382 |
| LOF | 0.3746 | 0.3745 |
| Matrix Profile | 0.2109 | 0.2112 |
| Rolling Z-Score | 0.1086 | 0.1083 |

这说明：

- IDKD 明显需要标准化。
- kNN 在这组中性裁剪的 12 条序列上变化较小，但它仍属于通常需要注意尺度的距离型方法。
- Isolation Forest 基本不受影响。
- LOF、Matrix Profile、Rolling Z-Score 在这批消融数据上变化不大。

---

## 10. 参数会不会影响结果

会，而且影响不小。

我们专门跑了参数敏感性实验：

- 12 条真实时间序列
- 20 个参数配置
- 240 条结果

每个算法中表现最好的配置：

| Algorithm | Best Config | Mean VUS-PR | Runtime |
|---|---|---:|---:|
| LOF | k=10 | 0.4039 | 0.21s |
| kNN | k=10 | 0.3797 | 0.21s |
| Isolation Forest | n=200, max_samples=512 | 0.3452 | 0.23s |
| IDKD | n=200, max_samples=16 | 0.3265 | 2.49s |

通俗解释：

- kNN 的邻居数不能太小，也不能太大。这里 `k=10` 比默认 `k=5` 更好。
- LOF 也依赖邻居数，`k=10` 在这批中性裁剪调参子集里平均分最高。
- Isolation Forest 的最佳配置是 `n=200,max_samples=512`，但主实验仍使用统一默认配置保证公平。
- IDKD 对 `max_samples` 很敏感，这轮是 `max_samples=16` 更好。

但注意：这些参数实验只是补充分析，不应该替代主实验。因为主实验要保持统一默认配置，避免“为每个算法调到最好再比较”带来的不公平。

---

## 11. 为什么有些算法会失败

不同算法有不同假设。

如果异常是尖峰，Rolling Z-Score 可能有效。

如果异常是形状不同，kNN 或 Matrix Profile 可能有效。

如果异常段处在低密度区域，LOF 可能有效。

如果异常容易被随机切分隔离，Isolation Forest 可能有效。

如果数据噪声很高，距离和密度结构会被破坏，kNN 和 LOF 可能下降。

如果异常持续很长，它可能形成一个新的“局部正常模式”，距离型方法就不一定能把它看成异常。

所以我们的研究不是简单说“某算法永远最好”，而是说：

> 算法表现取决于数据特征和算法机制是否匹配。

---

## 12. 当前最适合写进报告的结论

可以写成下面这样：

1. kNN 在 60 条真实单变量时间序列上取得最佳整体 VUS-PR。
2. Isolation Forest 在效果和运行时间之间最均衡。
3. LOF 在部分数据集上有竞争力，但对重复值和邻居数敏感。
4. IDKD 使用官方实现后更严格，但运行成本显著升高。
5. Matrix Profile 适合某些形状异常，但整体不是最强。
6. Rolling Z-Score 是快速但较弱的统计基线。
7. 标准化、窗口大小和参数选择都会影响结论。
8. 长序列裁剪必须不依赖异常标签，否则会引入评价泄漏风险；我们已经改为中性中心裁剪。
9. 数据特征会显著影响算法表现，因此解释性分析比单纯排名更重要。

---

## 13. 最终一句话

如果只能用一句话总结我们的项目：

> 我们发现，在统一的 range-aware VUS-PR 评价和不依赖标签的中性裁剪下，kNN 是最强的整体 baseline，Isolation Forest 是最均衡的实用选择，而不同算法的成败主要取决于时间序列异常的形状、比例、持续时间、噪声和参数设置。
