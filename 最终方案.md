# 时间序列异常检测算法比较：最终实施方案

## 1. 外部评价取舍

### 采纳的建议

- 强化 novelty（新意）：本研究不能只是复现 TSB-UAD / TimeEval 的 benchmark（基准评测），核心区别应改成“数据特征 -> 算法机制 -> 性能表现”的解释型分析。
- 增加 IDK / IK-based method：加入 Isolation Kernel family（隔离核家族）方法，因为作业材料明确推荐 Isolation Kernel 系列方法，也更贴合课程背景。
- 扩大数据集规模：原来的 20-40 条偏保守，修订为约 60 条 labeled univariate time series（带标签单变量时间序列），并要求来自多个 source datasets（来源数据集）。
- 明确 Matrix Profile 对齐方式：所有 window-level scores（窗口级异常分数）必须统一映射到 point-level scores（点级异常分数），避免不公平比较。
- 增加 controlled synthetic experiment（可控合成实验）：用少量合成序列支持 RQ4 的机制解释，避免“事后讲故事”。
- 加强页数控制：最终报告必须每个 RQ 对应少量核心图表，不能堆大表。

### 部分采纳的建议

- 数据集数量不盲目扩到 100-250 条；两人团队月底前完成，60 条左右更稳，重点保证 source diversity（来源多样性）和分析深度。
- 不把 anomaly type（异常类型）强行人工标注到所有真实数据上；真实数据用可计算特征分组，合成数据用明确的 point / contextual / collective anomalies。
- Algorithm family removal（算法家族移除）不作为主消融实验，而放入 discussion（讨论）中解释方法互补性。

### 不采纳的建议

- 不把 deep learning（深度学习）加入主实验；理由不是“做不完”，而是本研究 aim 强调可解释机制比较，深度模型受架构、训练和调参影响太大，不适合作为月底前的公平主比较。
- 不删 Rolling Z-Score；它是必要的 simple statistical baseline（简单统计基线），可以帮助说明复杂算法是否真的带来收益。
- 不把数据集缩到 15-20 条；这会削弱 assignment 中 “many datasets” 的高分要求。

## 2. 研究目标

**Aim Statement**

> The aim of this study is to evaluate and explain the accuracy, robustness, and computational efficiency of representative unsupervised anomaly detection algorithms for univariate time-series data, with a focus on how dataset characteristics and algorithm mechanisms jointly affect detection performance.

中文解释：本研究比较并解释多种无监督异常检测算法在单变量时间序列上的准确性、鲁棒性和效率，重点分析数据特征和算法机制如何共同影响检测表现。

**Research Questions**

- RQ1：哪些 unsupervised anomaly detection algorithms（无监督异常检测算法）在多数据集上整体表现最好？
- RQ2：算法表现是否受 anomaly ratio（异常比例）、series length（序列长度）、anomaly duration（异常持续时间）、seasonality（季节性）和 noise level（噪声水平）影响？
- RQ3：不同算法在 accuracy（准确性）和 runtime（运行时间）之间是否存在 trade-off（权衡）？
- RQ4：算法失败是否能由其 mechanism（机制）解释，例如 distance-based（基于距离）、density-based（基于密度）、isolation-based（基于隔离）或 subsequence-based（基于子序列）的假设？

## 3. 实验设计

### 数据集

- 主数据源使用 TSB-UAD Public-v2，UCR Time Series Anomaly Datasets 2021 作为可选 external validation（外部验证）。
- 目标规模为约 60 条 labeled univariate time series。
- 抽样规则必须 deterministic（确定性）：从至少 8 个 source datasets 中分层选取，覆盖短/中/长序列、低/中/高 anomaly ratio、短/长 anomaly duration。
- 不允许只挑表现好看的数据；报告中必须说明 sampling rule（抽样规则），防止 cherry-picking（选择性挑样本）。

### 主算法

- Rolling Z-Score：statistical baseline（统计基线）。
- kNN Detector：distance-based detector（基于距离的检测器）。
- LOF：Local Outlier Factor，density-based detector（基于局部密度的检测器）。
- Isolation Forest：tree/isolation-based detector（基于树和隔离机制的检测器）。
- IK-based Density Detector：使用 `ikpykit.anomaly.IDKD` 的官方 Isolation Distributional Kernel Detector 实现，作为课程相关方法。
- Matrix Profile：subsequence-based detector（基于子序列的时间序列专用检测器）。

### 辅助算法

- One-Class SVM 不作为主算法；若实现和运行时间允许，可作为 supplementary boundary-based detector（补充边界型检测器），但最终主结论不依赖它。

### 统一 Pipeline

1. Data loading（数据加载）：读取 values 和 anomaly labels。
2. Preprocessing（预处理）：缺失值处理、非法值处理、z-normalization（标准化）。
3. Sliding-window embedding（滑动窗口嵌入）：默认 `w = max(16, min(128, floor(0.02 * n)))`。
4. Model scoring（模型打分）：每个算法输出 window-level anomaly score。
5. Score alignment（分数对齐）：点 `t` 的分数取所有覆盖 `t` 的窗口分数最大值，即 `s_t = max(score_i)` for windows covering `t`。
6. Evaluation（评价）：labels 只在评价阶段使用，不参与训练或调参；主 VUS 指标使用官方 range-aware VUS 计算，近似 VUS 只作为 sanity-check 对照。
7. Analysis（分析）：按算法、指标、数据特征和 runtime 分组。

## 4. 评价指标与消融实验

### 主指标

- AUROC：整体排序能力。
- AUPRC：更适合 low anomaly ratio（低异常比例）场景。
- VUS-ROC：考虑时间序列范围容忍度的官方 range-aware ROC 指标。
- VUS-PR：考虑时间序列范围容忍度的官方 range-aware PR 指标，作为主要排序指标。
- Runtime：计算效率。
- Average Rank：跨数据集平均排名。

### Core Ablations（必做消融）

- Window Size Sensitivity：比较 1%、2%、5% series length 的窗口，验证窗口大小对方法的影响。
- Normalization Effect：比较 raw values（原始值）和 z-normalized values（标准化值）。
- Metric Choice Effect：比较 AUROC、AUPRC、VUS-PR 的排名差异，并用 Spearman / Kendall ranking correlation（排序相关性）可视化。
- Synthetic Mechanism Test：生成少量 point anomalies（点异常）、contextual anomalies（上下文异常）、collective/shape anomalies（群体/形态异常），验证算法机制解释是否站得住。

### Optional Ablations（选做）

- Thresholding Strategy：比较 best F1 threshold（最优阈值）和 contamination-based threshold（按异常比例设阈值）。
- Training Setting：若数据支持，比较 normal-only training（仅正常片段训练）和 full-series scoring（整序列打分）。

## 5. 分析与报告策略

### 核心分析故事线

- 先回答 RQ1：用 average rank 和主指标说明总体表现。
- 再回答 RQ2：按 anomaly ratio、duration、length、seasonality 分组，说明算法敏感性。
- 再回答 RQ3：用 Runtime vs AUPRC / VUS-PR scatter plot 展示 accuracy-efficiency trade-off。
- 最后回答 RQ4：用 2-3 个 case studies（案例分析）和 synthetic experiment 解释成功/失败原因。

### 必须产出的核心图表

- Overall performance ranking table。
- Metric ranking correlation heatmap。
- Runtime vs VUS-PR 或 Runtime vs AUPRC scatter plot。
- Performance by dataset characteristics 的分组柱状图或箱线图。
- Window size ablation plot。
- Normalization ablation plot。
- 2-3 张 time series + anomaly label + model score overlay 的案例图。

### 最终报告压缩原则

- 正文最多 10 页，不能把完整结果表堆进去。
- 每个 RQ 最多用 1-2 个核心图表支撑。
- 大量 per-dataset results（逐数据集结果）只作为附录或不放正文。
- Discussion 必须写 mechanism-based explanation（基于机制的解释），不能只写“算法 A 分数更高”。

- 主实验限定为 univariate time-series anomaly detection。
- 主设置为 unsupervised setting，labels 只用于评价。
- 主数据集为 TSB-UAD，UCR 只作为可选验证。
- 主算法固定为 Rolling Z-Score、kNN、LOF、Isolation Forest、IK-based Density Detector、Matrix Profile。
